# ADR: Bounded GPT-5.6 Employer Report Advisory

## Status

Accepted — 2026-07-17.

## Context

SignalLoop's scores, FAVO labels, integrity signals, and recommendations are generated by
deterministic code. Employers can still benefit from a short synthesis of what the captured process
evidence supports, what it does not establish, and what to ask in a follow-up interview.

Sending the complete evidence report to an external model is not acceptable. It contains hidden
test results, seeded issue areas, scoring internals, submitted code, integrity/proctoring details,
and evaluator-only material that the AI-provider boundary prohibits sharing.

## Decision

Add an optional GPT-5.6 advisory that runs only when `REPORT_ADVISORY_ENABLED=true` and an OpenAI
API key is configured.

The provider receives a newly constructed allowlist containing only assessment/timing metadata,
aggregate public-test results, candidate-test counts, aggregate AI prompt/redirect counts, existing
FAVO labels, candidate-authored submission-review text, and role coverage/caveats. The complete
report is never passed to the provider.

The allowlist excludes hidden tests, seeded issue areas, scoring/rubric internals, recommendations,
submitted code, evaluator notes, reference solutions, integrity details, and proctoring artifacts.

The response is limited to a summary, up to three evidence gaps, and up to three interview focus
areas. It is displayed as explicitly non-scoring and cannot modify scores, FAVO, integrity labels,
follow-up generation, or recommendations. Provider failures return `status=unavailable` and do not
block deterministic report generation.

## Consequences

- Employers get a concise advisory without making an LLM the evaluator of record.
- Deterministic reports remain usable when the feature is disabled or unavailable.
- Tests protect the allowlist and fail-open behavior.
- Any expansion of model inputs or influence requires a new safety review and ADR update.
