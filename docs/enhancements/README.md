# Enhancements

This directory contains post-MVP enhancement workstreams.

The original MVP phase plan under `docs/execution/phases/` is complete and should be
treated as historical implementation context. New work should start from
`CURRENT_STATE.md` and the active enhancement workstream below.

## Active Workstream

- `phase-2-assessment-system/` - Assessment System Enhancement.

## Rules

- Do not use enhancement docs as permission to skip bounded implementation tasks.
- Keep changes one workstream step at a time.
- Update `CURRENT_STATE.md` after each completed enhancement task.
- Add ADRs when implementation changes architecture, scoring, AI boundaries, or external services.
